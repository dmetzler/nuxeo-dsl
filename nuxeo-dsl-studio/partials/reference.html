Doctype Declaration

            
doctype <doctype name> extends <super type>{
    schemas {
        <schema name> [{ <property> [<type>], ...} ]
    }

    aliases {
        <alias name> <aliastype> { <alias arguments>}
    }

    [crud]
}

  • <doctype name> is the name of the document type to be declared,

  • <super type> is the parent type of this document type (document by default),

  • <schema name> is the list of schema name that this document follows. It's possible to declare inline schemas,

  • <alias name> is the name of the aliased property,

  • <aliastype> is the type of alias used that cane be (prop|query),

  • <alias arguments> is the comma separated list of arguments for the alias. ,

  • and as an option <crud> asks to generate mutations for the type.

The possible arguments for alias depends of the type of the alias.

  • for <prop> it is the name of the pointed propertym for instance dc:title,

  • for <query> it is the nxql query that resolve the property, followed optionally by the type of the items returned by the query,

Schemas Declaration

            
schema <schema name> {
    <property name> [<property type>],
    <property name> [<property type>]
}

  • <schema name> is the name of the schema,

  • <property name> type of the property. If ommitted it's string by default.

Queries Declaration

            
queries  {
<query name>[(<params>)][:<return type>] "<nxql>"
}

  • <query name>is the name of the query as it will be used in GraphQL

  • <params> is a comma separated list of parameter names,

  • <return type> is type of the item returned by the query (by default document,

  • <nxql> is the Nuxeo query expressed in NXQL.
